# Security policy

Do not open public issues with secrets, credentials, private customer data, production configs, or sensitive incidents.

For security concerns, use this placeholder contact:

https://example.com/security

## Before sharing examples

Make sure examples are synthetic or fully anonymized.

## Agent safety

Agents should not receive unrestricted access to:

- customer communications;
- payment systems;
- production databases;
- credentials;
- private legal/financial data;
- publishing channels.

Use permissions and approvals first.
