# Department Rollout Roadmap

Company Brain System rolls out one department at a time. Dirección is mandatory first; every later department gets a brain, owner, digital employee pack, skills, approvals and receipts.

## 48 hours — establish control

Goal: create a safe operating base and one useful digital employee.

- Dirección / Cerebro Madre created.
- Company type selected: agency, consultancy or freelancer.
- Approval boundaries written.
- Accountability map drafted.
- First digital employee launched: Dirección Assistant or CEO / Operations Assistant.
- First safe task completed with receipt.

## 7 days — build the service operating system

Goal: make the core service delivery visible and repeatable.

Recommended order:

1. Dirección / Mother Brain.
2. Operations / Delivery.
3. Marketing.
4. Sales.
5. Customer Success / Postventa.

Outputs:

- department brains for selected departments;
- one human owner per department;
- agent onboarding pack per department;
- skills per department;
- weekly cadence and scorecard;
- issue list and improvement backlog;
- receipts/statechanges/context packets in use.

## 30 days — become AI-First, not just AI-assisted

Goal: install a feedback loop across departments.

Add:

- Product / Software Internal Systems.
- Finance.
- People / Organization.
- Admin / Legal / Compliance where needed.
- trace policy and model strategy;
- feedback loop: work → receipt → human correction → skill/template update → statechange.

## Department definition of done

A department is installed only when it has:

- `department-brain.md`;
- human owner and accountability seat;
- digital employee pack;
- allowed tools and forbidden actions;
- skills list;
- SOP list;
- scorecard metrics;
- escalation rules;
- first receipt.

## Rollout principles

- People before agents.
- Memory before automation.
- Receipts before claims of completion.
- Small safe tasks before production changes.
- External/public/economic/legal/sensitive actions require human approval.
