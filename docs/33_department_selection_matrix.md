# 33 — Department Selection Matrix

Do not install every department on day one unless the company already has enough maturity and owners. Start with Dirección / Mother Brain, then add the smallest set of departments that removes the current bottleneck.

## Inputs

- Company type: `freelancer`, `agency`, `consultancy`.
- Team size: `1`, `2–5`, `6–15`, `16+`.
- Maturity: `chaos`, `documented`, `assisted`, `ai-first`.
- Primary bottleneck: acquisition, sales, delivery, quality, retention, cashflow, hiring, compliance.

## Default order

1. Dirección / Mother Brain.
2. Operations / Delivery.
3. Marketing.
4. Sales.
5. Customer Success / Postventa.
6. Product / Software internal systems.
7. Finance.
8. People / Organization.
9. Admin / Legal / Compliance.

## Matrix

### Freelancer

- Team size `1`, maturity `chaos`: install Dirección now; defer departments except Operations/Delivery and Finance hygiene.
- Team size `1`, maturity `documented`: install Dirección + Operations/Delivery + Sales if proposals are repeated.
- Team size `2–5`: install Dirección + Operations/Delivery + Sales + Finance; add Marketing if acquisition is the bottleneck.
- Assisted or AI-first: add Customer Success and Product/Software only if recurring clients or internal assets exist.

### Agency

- Team size `2–5`, chaos: install Dirección + Operations/Delivery; add Sales only after approvals are clear.
- Team size `6–15`, documented: install Dirección + Operations/Delivery + Sales + Marketing + Customer Success.
- Team size `16+`: add Finance and People early because capacity, margin and accountability become system constraints.
- AI-first: add Product/Software and Admin/Legal/Compliance for tools, experiments and risk review.

### Consultancy

- Team size `1–5`: install Dirección + Operations/Delivery + Meeting / Context Packet Assistant.
- Team size `6–15`: add Sales, Customer Success and Finance for pipeline, renewals and margin review.
- Team size `16+`: add People and Admin/Legal/Compliance before scaling digital employees across partners.
- AI-first: add Product/Software to turn repeatable expertise into assets, tools or productized IP.

## Do-not-install-yet rules

- Do not install People before roles and responsibilities exist.
- Do not install Admin/Legal/Compliance as an active agent without explicit approval boundaries.
- Do not give any department production/client write access on day one.
- Do not install Product/Software before there is a clear internal system or product backlog.

## Output format

For each company, produce:

- departments now;
- departments next;
- departments later;
- explicit deferrals;
- first digital employee;
- required receipts.
