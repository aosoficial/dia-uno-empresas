# Human SOPs — how people operate Company Brain System

## SOP 1 — Before asking an agent to work

1. State the goal.
2. Point to the source of truth.
3. Say what is forbidden.
4. Say what requires approval.
5. Define the expected output.
6. Ask the agent to list missing context before acting.

## SOP 2 — When the agent asks a question

Answer only what is needed.

If the agent asks too broadly, ask it to use 1:3:1:

- what is the problem;
- what are three possible solutions;
- what does it recommend?

## SOP 3 — When approving work

Before approval, check:

- Is the context correct?
- Are assumptions labeled?
- Is private data protected?
- Is the recommended action reversible?
- Is there evidence?
- Is the next step clear?

## SOP 4 — Weekly review

Every week, review:

- decisions made;
- receipts created;
- blockers;
- missing context;
- repeated questions;
- permission issues;
- what should become a template.

## SOP 5 — Adding real data

Do not add real data until:

- data types are classified;
- owners are assigned;
- permissions are written;
- retention rules are clear;
- approval gates exist;
- access is limited.

## SOP 6 — When to ask Libera

Ask for help when:

- agents keep asking the same questions;
- the team cannot define permissions;
- memory becomes messy;
- receipts are not useful;
- no one reviews the system;
- adoption fails;
- the system touches sensitive operations.
