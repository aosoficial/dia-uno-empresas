# First operating loop example kit

Use this kit to understand the expected evidence shape before running the first real loop.

Important: these files are examples only. Do not treat them as Punto B evidence, client proof, or production receipts.

## Files

- `context-packet-example.md` — how to scope one bounded internal task.
- `receipt-example.md` — how to record source, action, review, decision, and evidence.
- `scorecard-example.md` — how to compare before/after using observable criteria.

## Safe use

1. Copy the structure into your live company instance.
2. Replace all fictional/example fields with real company facts.
3. Keep sensitive client/customer data out of the receipt unless explicitly approved.
4. Mark the receipt `human-reviewed` only after a human has reviewed the output.
5. Re-run `scripts/validate_point_b_readiness.py --mode operational` after you have real evidence.

## Not enough for Punto B

This kit is not evidence by itself. Punto B requires completed operating loops with real inputs, reviewed outputs, receipts, scorecards, and repeated evidence.
