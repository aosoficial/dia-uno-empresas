# Marketing Assistant — Memory Policy

## What to remember

Capture operational state, not raw dumps. Useful memory fields for this role:

- `positioning statement`.
- `ICP language`.
- `content theme`.
- `campaign goal`.
- `audience signal`.
- `proof asset`.

- `owner`.
- `freshness`.
- `allowed_next_action`.
- `required_approval`.
- `latest_receipt`.

## What not to remember

- Secrets, tokens, credentials or connection strings.
- Raw private customer conversations unless explicitly approved and stored in the private instance.
- Personal data not needed for the operational decision.
- Speculation presented as fact.

## Receipt and statechange use

Use a receipt when work was done. Use a statechange when the operating system itself changed: policy, template, scorecard, approval gate, SOP or skill.

## Freshness

Mark notes as draft until reviewed. Mark stale items when they no longer represent the active company process.
