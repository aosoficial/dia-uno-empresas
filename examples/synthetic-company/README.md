# Synthetic example — Meridian Studio

This is a fake company. Do not replace it with real customer data until privacy rules are defined.

## Company

Meridian Studio sells design and implementation projects to B2B clients.

## Problem

The founder remembers too many decisions manually. Proposals, client requests, and delivery commitments are scattered across calls, documents, and chat.

## First Company Brain scope

Only sales-to-delivery handoff.

## First agent

Atlas, an internal operations agent.

Allowed:

- summarize synthetic notes;
- create context packets;
- draft internal task lists;
- write receipts.

Forbidden:

- contact customers;
- change prices;
- publish;
- use real private data;
- make commitments.

## First task

Create a context packet for a fake client handoff and identify missing information before delivery starts.
